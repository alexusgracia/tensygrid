District Heating Pipe - Object based implementation

In a district heating network, where a high number of pipes (and other component) exist, generating equations for each pipe individually is not efficient. For this purpose, object oriented approach based on classes are generated
(For the basic implementation of a district heating pipe in the MTI-framework, please refer to Basic Implementation )

Within the context of this modelling framework, the direction of the mass flow is represented using a boolean variable , which is later used to generate the absolute value of the mass flow .
If multiple "pipes", with its own boolean variable , are connected to each other, the simulation breaks, when the direction of the mass flow changes. This occurs due to multiple switching events (inequalities) triggering at exactly the same time. This constillation may cause Zeno behavior (or also chattering) to occurs leading to no results being found for the discrete events.
This can be avoided by applying a single boolean variable for every pipe with identical mass flow. In other words for every pipe connected in a row.
This is implemented using an object called FluidDirectionBool.

];
